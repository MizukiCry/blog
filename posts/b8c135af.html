GSoC 2023 Diary

发表于2023-02-24|更新于2023-03-15
|浏览量:

Intro

现在已经连续两个学期学分都只有 20 左右了，然而周围同学基本都是冲着 30 分去的，时间比较充裕就打算尝试一下开源工作

去年观望了一年 GSoC，今年年初又错过了 LFX Mentorship，想了想还是冲一次今年的 GSoC，总不能一直颓废下去

2023-2-23

今天 GSoC 的 Organizations 公布了，大概看了一下，大部分技能要求似乎都不是 C++ 或者 Rust 这种偏底层的，像 Apache，GNU 社区之类的感觉又太硬核了（？）

想起来之前的 LFX Mentorship 看见有个叫做 WasmEdge 的项目有 C++ 相关的 Task，找了找在 CNCF 下面，顺着研究了一下又看到一个叫 Curve 的项目，三个 Ideas 也是 C++ 方面的

Curve 似乎是网易的一个开源项目，A could-native distributed storage system。虽然我对分布式存储不是很了解，但是其中一个 Task 是给快照实现数据压缩功能，比较的独立，决定先联系一下试试。

2023-2-24

昨晚上在 Issues 下面问了之后，第二天就拉了个微信群。

开始研究快照部分的源码，不过看了看不是很明白

2023-2-25

开始着手写 Proposal

2023-2-27

去微信群里问了问 Proposal 的格式，不过没人回复🥲，难道问题太弱智了吗（

2023-2-28

先不管这么多了，开始着手写 Proposal。网上找了几份之前 GSoC 选手的 Proposal 开始照着抄

把之前写的一些传到 Github 上的简单代码拉来凑数。

2023-3-3

打算试试 Github 的 Pull Request 功能，就去提了个 fix typo 的 pr

2023-3-6

完成了初步的 Proposal，于是发到微信群里请 mentor 给点意见，但是还是没回复我（

感觉怪没动力的

同时 Github 的 Issues 下面开始出现了第二位对这个 Task 感兴趣的人，来压力了（

2023-3-11

Issues 下出现了第三位候选人，看了看是个印度老哥

。。。说实话对阿三印象不太好，看了看往年的 GSoC 申请情况过于卷了，GSoC-CN 群里印象普遍也一般，经常有抢 Issue 的情况

不过印度人到真的挺自信的，profile 一个比一个花哨，Issues 下面连番轰炸

然后我看了看这人也提了个修文档的 pr，感觉。。。不好说

2023-3-14

mentor 可算出来说了几句，感觉大体的方向应该没错，不过代码还不是很懂，得再研究探究，最近摸鱼太严重，留了一堆事没做，急
